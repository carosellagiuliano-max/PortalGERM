# SwissTalentHub / PortalGERM

Dieses Repository enthält aktuell den vollständig übertragenen und auditierten **Implementierungsplan** für SwissTalentHub. Es enthält **noch keinen** Anwendungscode, kein Datenbankschema, keine funktionierenden Mock-Provider und keine Tests.

Einstieg:

1. [`AGENTS.md`](./AGENTS.md) — Implementierungs- und Evidence-Regeln.
2. [`codex-plan/00-PLAN.md`](./codex-plan/00-PLAN.md) — Masterplan und aktueller Status.
3. [`codex-plan/product-strategy.md`](./codex-plan/product-strategy.md) — Positionierung, Journeys, Marketplace, Growth und Monetarisierung.
4. [`codex-plan/architecture-blueprint.md`](./codex-plan/architecture-blueprint.md) — Routen, Rollen, Daten, Security, UX, Tests und Betrieb.
5. [`codex-plan/requirements-matrix.md`](./codex-plan/requirements-matrix.md) — Traceability der Anforderungen.
6. [`codex-plan/implementation-plan.md`](./codex-plan/implementation-plan.md) — ausführbare Schritte 01–18.
7. [`codex-plan/repository-audit.md`](./codex-plan/repository-audit.md) und [`codex-plan/plan-audit.md`](./codex-plan/plan-audit.md) — verifizierter Ist-Zustand, Transfer, Konflikte und offene Entscheidungen.

Die Implementierung beginnt mit Phase 01. Kein aus dem Quellrepository geerbtes Häkchen ist ein Nachweis für dieses Zielrepository.

Das geplante MVP nutzt persistente lokale Mock-Provider und wird nur als datenschutzfreundlich vorbereitetes, kontrolliertes MVP beschrieben. Solange Rechts-, Steuer-, Provider- und Betriebsprüfungen offen sind, darf es nicht als produktionsbereit oder vollständig DSG-konform bezeichnet werden.
